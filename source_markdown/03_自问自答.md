## 整理正文

## 1. LLM 方法使用什么大模型？

本研究选用了 **Llama 3.3 8B-Instruct** 和 **GPT-3.5-Turbo** 作为测试模型。

选择理由：
- 兼顾成本与可复现性
- 直接对应 DrSR 和 LLMSR 原论文中的实验设置
- 能保证对比的公平性和相关性

## 2. 为什么不能在 800 个数据集上做全量超参数搜索？

因为算力不够，无法对全量数据集做完整调参。

当前设定里：
- 总共有大约 800 个数据集
- 方法数约为 10 种
- 包含 LLM 方法、强化学习方法、传统方法等不同范式
- 大模型方法本身不一定需要大量传统意义上的超参数搜索

因此需要从 800 个数据集中，挑选一个 **调参核心集**，先在核心集上完成超参数搜索，再把最优参数冻结后作用到 800 个数据集上。

## 3. 如何科学地精选调参核心集？

提出的方法论名称是：

**Purpose-Driven Stratified Sampling（目的驱动的分层抽样）**

目标不是随机抽样，而是构建一个能代表 800 个数据集全部“挑战轴”的 **微缩景观（microcosm）**。

### 3.1 最终目标
- 选出 25 个数据集，构成 **Tuning Core Set**
- 这个集合需要在论文中能够被清晰辩护

### 3.2 第一步：定义四大支柱与三大挑战轴

#### 四大支柱（数据来源）
- SRBench（经典集）
- SRSD（真实/噪声集）
- LLM-SRBench（现代/LLM 集）
- SDC-100（你自己的深度集）

#### 三大挑战轴（难度维度）
- 维度：低维 vs 高维
- 噪声：纯净 vs 有噪
- 难度：简单 / 中等 / 困难

### 3.3 第二步：定义“难度”

#### 对于 SDC-100
- 难度使用 AST 深度定义
- 可以按 L1 / L2 / L3 / L4 / L5 分层

#### 对于其他约 700 个数据集
采用实证探测法：
- 用一个廉价算法（如 gplearn 或 PySR）
- 设置极短超时（例如 30 秒）
- 用实测结果定义简单、中等、困难

定义示例：
- 简单：30 秒内完美求解，`R^2 > 0.999`
- 困难：30 秒内 `R^2 < 0.9`
- 中等：介于两者之间

### 3.4 第三步：按挑战目的分配 25 个席位

#### SDC-100：5 个
- 这是你自己的核心创新数据集
- 需要过采样，保证 HPS 过程看见并认真对待这个挑战
- 从五个深度等级各选 1 个

#### SRSD：6 个
- 用于测试噪声鲁棒性与真实世界拟合能力
- 采用 3 x 2 矩阵：
  - 简单 / 中等 / 困难
  - 低噪 / 高噪

#### SRBench：8 个
- 覆盖经典的 Feynman 和 PMLB 问题
- 既保留社区共识，也保留黑盒难题

#### LLM-SRBench：6 个
- 用于测试组合性与语义理解挑战
- 覆盖简单 / 中等 / 困难、低维 / 高维

### 3.5 第四步：如何在论文里捍卫这个选择？

核心话术是：
- 在计算可行性与评估公平性之间取得平衡
- 参考 SRBench 2.0 / SRBench++ 的最佳实践
- 通过目的驱动的分层抽样构建调参核心集
- 保证四大数据来源和三大挑战轴均得到充分代表

## 4. 核心数据集名称怎么起？

### SSR-50
- 全称：**Scientific Symbolic Regression**
- 来源：从 800 个数据集实验之后筛出来的 50 个子数据集
- 作用：作为以后新算法比较的高价值子集，不必每次都直接跑满 800 个数据集

### SR-Cal-25
- 全称：**Symbolic Regression Calibration Set**
- 作用：专门用于调参
- 要求：必须与 SSR-50 不相交，避免数据泄露

使用流程：
1. 新算法先在 SR-Cal-25 上找到最优参数
2. 然后再去 SSR-50 上测试

## 5. 为什么选这 10 个算法？

你把 benchmark 方法分成了 5 个主要技术范式，每类 2 个方法。

### 5.1 大语言模型方法（2）
- LLM-SR：代码生成式 LLM-SR 的开创者
- DrSR：引入双重推理机制的改进版

### 5.2 遗传算法（2）
- PySR：现代 GP 标杆
- Operon：高性能 C++ GP 实现

### 5.3 深度强化学习（2）
- DSO：早期深度 RL 符号回归方法
- uDSR：统一框架

### 5.4 预训练 Transformer（2）
- E2E：端到端 Transformer
- RAG-SR：检索增强混合预训练方法

### 5.5 树搜索方法（2）
- iMCTS：改进 MCTS
- TPSR：Transformer + MCTS 规划

## 6. 如果审稿人质疑分类怎么办？

你的回应逻辑是：

- 承认某些方法属于混合范式
- 但分类依据不是“使用了哪些技术”，而是“核心创新点在哪里”

例如：
- uDSR：核心贡献是统一强化学习框架
- TPSR：核心创新是树搜索规划机制
- RAG-SR：核心独特性在于检索增强

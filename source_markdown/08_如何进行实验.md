2. 整个流程围绕 Redis + Celery + Flower + 多 Worker 节点展开。

## 整理正文

## 如何结束任务
```bash
celery -A tasks purge -Q cpu_queue
bash stop_all_workers.sh
```

## 阶段一：基础服务启动

### 1. 启动 Redis（消息代理）
使用系统级 Redis。

```bash
redis-cli ping && echo "  ✓ Redis 运行正常，可以直接使用" || echo "✗ Redis 有问题"
```

### 2. 启动 Flower（监控面板）

进入项目目录并激活环境后，启动 Flower：

```bash
# celery -A tasks flower --address=0.0.0.0 --port=5555
tmux new-session -d -s celery "celery -A tasks flower --address=0.0.0.0 --port=5555"
```

说明：
- Flower 面板运行在 `http://10.5.40.168:5555`
- 对当前 PySR 探针流程而言，FastAPI 服务不是必须的

## 阶段二：每次运行探针的标准顺序

### 步骤 1：部署并启动计算节点（Workers）

```bash
bash deploy_workers.sh
```

作用：
1. 将最新代码（如 `tasks.py`, `run_single_pysr.py`）拷贝到所有 Worker 节点
2. 在所有 Worker 上安装 / 更新 `requirements.txt`
3. 在所有 Worker 上建立反向 SSH 隧道
4. 启动 tmux 会话运行 Celery Worker 进程
5. Worker 自动连接到 Redis，并在 Flower 中显示为 Online

### 步骤 2：提交探针任务（Client）

```bash
python submit_benchmark.py
```

作用：
1. 扫描 `~/.sim_datasets/` 目录
2. 找到全部数据集
3. 循环创建 `run_pysr_probe` 任务
4. 将任务发送到 Redis 队列中

### 步骤 3：监控（Monitor）

在浏览器打开：

```text
http://10.5.40.168:5555
```

作用：
- Worker 从 Redis 领取任务
- 任务状态变为 Running
- 可以实时查看成功和失败的任务

### 步骤 4：收集结果（Collector）

等待所有任务完成后执行：

```bash
python collect_results.py
```

作用：
1. 从 Redis 中拉取每一个任务的返回结果（JSON）
2. 汇总保存到 `pysr_probe_results.csv`
3. 打印初步难度分析

# 实验设置、成本与指标

## 你的实验设计已经进入“可执行”阶段

从实验 setting 文档来看，你已经不只是提出框架，而是在量化：

- 每种方法的时间成本
- token 成本
- prompt 变量
- seed 数量
- 超参数搜索空间
- 全量实验规模
- 最终图表产出

这意味着项目已经从“想法阶段”走向“实验生产阶段”。

## 成本意识很重要

你已经在记录：
- DrSR 的运行时长、iteration 和 token 累计
- LLMSR 的运行时长、iteration 和 token 累计
- 对不同 prompt / background 设置造成的实验量级有清晰估算

这是后续论文里非常能加分的一点，因为很多 benchmark 只报告结果，不报告成本。

## 你的实验变量控制越来越清晰

### 对 LLM 方法
你已经在考虑：
- 不同背景提示
- 是否提供变量语义
- 是否匿名变量
- 是否强调真实物理意义
- prompt 模板的层次化设计

### 对传统方法
你已经在考虑：
- grid search
- seed 固定
- 冻结超参数后再全量跑

这让 LLM 方法和传统方法之间的比较更公平。

## 指标设计也在同步升级

你已经开始考虑三类指标：

### 1. 精度类
- `R^2`
- NMSE
- 恢复率

### 2. 结构类
- NED
- symbolic accuracy（SA）
- 表达式等价性评测

### 3. 效率类
- Time-to-Solution

### 4. 鲁棒性类
- 去除 top 5% 极端点之后的拟合表现

## 一个很强的信号

这些指标放在一起，说明你已经不把 benchmark 当成“单指标排名表”，而是当成一个多维能力画像系统。

## 图表产出也已经成型

你已经明确计划输出：
- 求解率
- 五分类求解率
- 求解效率
- `R^2`
- NMSE 随 iterations 的变化曲线

这说明你后面论文中的可视化主干已经有雏形。

## 后续最值得继续加强的点

### 1. 指标定义写成标准表
建议后面单独做一张总表，列出：
- 指标名称
- 记号
- 计算方式
- 反映能力
- 是否越高越好

### 2. 成本报告标准化
建议每次实验固定输出：
- wall-clock time
- API token
- approximate cost
- average time per iteration
- success / failure status

### 3. 结果表分层
建议最后论文里把结果拆成三层：
- 全量平均结果
- SSR-50 平均结果
- SSR-50 最佳结果

这样逻辑会更清楚。

# 全量探针实验
涉及到 664 个数据集，每个数据集用pysr 和 llmsr


## 候选 benchmark 种子筛选

在 `664` 个数据集上完成 `pysr` 与 `llmsr` 的双探针之后，进一步从中筛出 `200` 个候选种子，作为后续正式实验的候选池。

### 候选池的目标

- 用 `pysr` / `llmsr` 的探针结果先筛掉区分度低的数据集
- 保留方法偏好明显、单边可评估、以及中等 gap 但仍有信息量的样本
- 为后续正式实验提供一个规模可控、结构更均衡的 `200` 数据集候选池

### 最终采用的约束

- `subgroup <= 25`
- 同一 `basename` 的变体最多 `2` 个
- `srsd` 总数不超过 `70`
- 候选池总数固定为 `200`

### 最终 200 个候选的池子构成

- `池 A = 140`
  - 高 gap 主体
- `池 B = 20`
  - 单边可评估样本
- `池 C = 40`
  - 中 gap 且至少一方还有有效表现的样本

### 两种方法的优势配比

最终候选池中：

- `PySR` 优势：`120`
- `LLM-SR` 优势：`80`

### 四类标签摘要

为了后续分析更清楚，给这 `200` 个候选额外打了四类标签：

- `strict`
  - 池 A 中未放松 gap 下限的高区分度样本
- `relaxed`
  - 为了把 `LLM-SR : PySR` 优势配比从 `66:134` 调整到 `80:120`，在池 A 中额外补入的 `14` 个 `LLM-SR` 优势中低 gap 样本
- `one-sided`
  - 池 B 中单边可评估样本
- `mid-gap`
  - 池 C 中双方都有指标、且至少一方仍有有效表现的中等 gap 样本

对应数量是：

- `strict = 126`
- `relaxed = 14`
- `one-sided = 20`
- `mid-gap = 40`

### 当前这版 200 候选池的解释

这版 `200` 个更适合作为：

- 正式实验的候选池

而不是：

- 最终论文里直接固定不变的 benchmark 定稿

原因是：

- 其中 `strict` 是高区分度主体
- `one-sided` 反映的是 failure-mode / 可用性差异
- `mid-gap` 负责补充“不是纯爆炸、但方法偏好仍然明显”的样本
- `relaxed` 则是出于方法配比平衡，主动加入的定向样本

因此，这 `200` 个的用途更适合：

1. 先做下游正式实验
2. 再结合更多算法的结果，把它进一步压缩成最终核心 benchmark 集

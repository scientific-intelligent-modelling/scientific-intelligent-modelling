\section{Introduction}

Symbolic regression (SR) aims to recover compact, interpretable mathematical expressions from data and remains central to scientific machine learning and scientific discovery. The SR ecosystem now spans early scientific-discovery systems and physics-inspired search \citep{schmidt2009distilling,udrescu2020aifeynman,udrescu2020aifeynman2}, policy-gradient and neural-guided approaches \citep{petersen2021deep,mundhenk2021seeding}, pretrained neural and transformer-based systems \citep{biggio2021nsr,valipour2021symbolicgpt,kamienny2022end,vastl2022symformer,tpsr2023}, practical software stacks such as PySR \citep{cranmer2023pysr}, and more recently large-language-model-assisted approaches \citep{shojaee2024llm,wang2025drsr}. However, research iteration is still slowed by tooling fragmentation as much as by modeling difficulty.

A typical SR method is released as a standalone repository with its own environment assumptions, data conventions, execution scripts, and result formats. Different benchmark families also expose incompatible file layouts, split conventions, metadata, OOD definitions, and ground-truth annotations. As a result, a substantial fraction of SR effort is spent on repository repair, environment isolation, data reshaping, evaluation alignment, and result auditing before scientific comparison can even begin.

This paper argues that symbolic regression needs a unified \emph{research substrate}: a system that makes heterogeneous methods executable under a shared protocol, makes heterogeneous datasets consumable under a shared schema, and exposes the resulting workflow to structured automation. To this end, we present \textbf{SREvoLab}, a lightweight auto-review and evolution framework designed to reduce the cost of method onboarding, dataset canonicalization, reproducible execution, and bounded iterative improvement. SREvoLab is built around two normalization layers. The first is a \emph{unified method interface}: representative SR systems are wrapped behind a common execution contract for training, prediction, and equation extraction, while incompatible dependencies are isolated by environment management. The current platform integrates ten heterogeneous SR backends spanning classical, search-based, and LLM-assisted approaches. The second is a \emph{canonical dataset schema}: heterogeneous SRBench-, LLM-SRBench-, and SRSD-style sources are reorganized into a lightweight format with \texttt{train.csv}, \texttt{valid.csv}, \texttt{id\_test.csv}, \texttt{ood\_test.csv}, \texttt{metadata.yaml}, and optional ground-truth formulas.

On top of this substrate, we introduce a modular \emph{skill layer}. Rather than leaving repository integration, data conversion, protocol checking, and expression processing as ad hoc chores, SREvoLab packages them into reusable operations such as repository onboarding, heterogeneous-data conversion, expression normalization, and formula-grounded validation.

Finally, we outline a lightweight \emph{audited evolution loop}. The loop starts from a runnable baseline, compiles a protocol-preserving experiment specification, executes the run, reviews the resulting artifacts, and only then proposes limited follow-up modifications. The emphasis is disciplined iteration under fixed benchmark semantics rather than unconstrained autonomous search. The key design choice is \emph{review before evolve}: evolution is driven by structured feedback over logs, equations, metrics, and audit flags, not by unchecked score chasing. In the current draft, this claim is supported by three concrete assets: a $10$-tool/$8$-environment method substrate, formula-grounded dataset validation with NMSEs down to $10^{-15}$ on canonicalized examples, and a pilot slice that exposes materially different ID/OOD behavior under one result format.

The main contributions of this work are as follows:
\begin{itemize}[leftmargin=1.5em]
    \item We build a unified symbolic regression substrate that integrates ten heterogeneous SR tools across eight controlled execution environments under one wrapper interface.
    \item We propose a lightweight canonical dataset schema with explicit train/validation/ID-test/OOD-test splits, metadata, and optional ground-truth formulas, and we verify formula consistency numerically when analytical targets are available.
    \item We develop a modular skill layer for repository onboarding, dataset homogenization, and formula-aware validation, turning recurring SR engineering work into explicit, reusable operations.
    \item We introduce a review-first audited evolution loop that maps run artifacts into bounded next-step proposals while preserving benchmark semantics and evaluator integrity.
\end{itemize}

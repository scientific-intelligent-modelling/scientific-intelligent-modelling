Symbolic regression research is increasingly limited by operational fragmentation rather than by a lack of methods alone. Symbolic regression systems are typically released as standalone repositories with incompatible environments, execution assumptions, data layouts, and expression formats, making it difficult to reproduce, compare, and extend them under a shared protocol. We present \textbf{SR-Workbench}, a unified research substrate for symbolic regression. SR-Workbench currently wraps $10$ heterogeneous symbolic regression backends under a common execution interface, isolates incompatible dependencies through environment management, and canonicalizes multiple dataset families into a lightweight schema with train/validation/ID-test/OOD-test splits, metadata, and optional ground-truth formulas.

On top of this substrate, we expose modular skills for repository onboarding, dataset homogenization, protocol compilation, formula-aware validation, and bounded iterative improvement. These components support a lightweight audited evolution loop that can reproduce baselines, align outputs, detect protocol violations, and propose constrained follow-up modifications without changing benchmark semantics. We evaluate the current system as infrastructure rather than as a new symbolic regression algorithm. Repository-archived pilot runs already provide aligned ID/OOD summaries across multiple methods and datasets, while formula validation confirms that declared ground-truth equations numerically match canonicalized datasets when available. SR-Workbench provides a practical foundation for reproducible symbolic regression experimentation and future SR-specific research automation.

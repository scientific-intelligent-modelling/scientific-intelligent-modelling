# 源数据映射模式

## 模式一：单个 CSV

适用：

- 一张总表，包含所有样本
- 可能还没切 train/valid/test

做法：

1. 确定目标列。
2. 过滤仅用于建模的数值列。
3. 设计 split 规则：
   - 随机切分
   - 时间切分
   - 实体分组切分
   - 范围外切分作为 OOD
4. 导出四个 CSV。
5. 若数据天然没有 OOD 且没有真值公式，可额外尝试：
   - 用逐维区间内缩法从总表中抽取 OOD
   - 目标是让 OOD 占比约 10%
   - 若达到目标前 OOD 占比超过 20%，则视为抽取失败，不生成 OOD

## 模式二：多文件已分好 split

适用：

- 用户直接给了 train/test 或多个 split 文件

做法：

1. 检查列头是否一致。
2. 若没有 `valid` 或 `ood_test`，按已有逻辑补切。
3. 统一列名和数值类型。

## 模式三：Excel / 多 sheet

适用：

- 一个 xlsx 中多个 sheet 表示不同 split 或不同实验批次

做法：

1. 先列出每个 sheet 的语义。
2. 再决定是：
   - 一个 sheet 对应一个 split
   - 还是多 sheet 合并后再切 split
3. 导出为 CSV 后再进入统一流程。

## 模式四：JSON / JSONL / 嵌套结构

适用：

- 原始数据不是平表

做法：

1. 先展平为二维表。
2. 明确保留哪些字段作为特征。
3. 丢弃仅用于标识、日志、调试的字段。
4. 再切 split。

## 模式五：时间序列 / 实体轨迹

适用：

- 有时间顺序
- 有多个 entity / run / subject

做法：

1. 先决定样本粒度：
   - 单时刻
   - 窗口
   - 差分
2. OOD 优先用：
   - 更晚时间段
   - 未见实体
   - 更极端取值范围
3. 若随机切分会泄漏，禁止随机切。

## 列处理建议

- 尽量保留有物理含义的原始列名。
- 若列名重复、含空格、单位太乱，可清洗成稳定列名。
- 若做了重命名，metadata 中必须写明原始含义。
- 非数值列默认不进入建模列，除非你显式做了合理编码。

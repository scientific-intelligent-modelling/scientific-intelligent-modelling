# Scientific Intelligent Modelling 使用教程（快速部署版）

本教程覆盖：本机快速上手 + check 脚本 + Docker 打包部署。

---

## 1. 仓库结构与前置条件

### 运行环境

- Python：3.10（主控环境）
- Conda：建议已安装（用于子环境隔离）
- Git：支持子模块
- Docker：用于打包部署（可选）

### 仓库准备

```bash
# 建议拉取子模块

git clone --recursive https://github.com/scientific-intelligent-modelling/scientific-intelligent-modelling.git
cd scientific-intelligent-modelling
```

> 如果已经有仓库且子模块未拉取：`git submodule update --init --recursive`
>
> 当前本地目录约定：
> - `sim-datasets-py/`：`sim-datasets` Python 包源码目录
> - `sim-datasets-data/`：本地数据集仓库目录
> - 若使用 CLI 传相对数据集路径，建议设置 `SIM_DATASETS_PATH=$(pwd)/sim-datasets-data`

---

## 2. 主机端安装（最小）

```bash
# 1) 创建主环境
conda env create -f environment.yml
conda activate sim

# 2) 安装主控包
pip install -e .

# 3) 按需创建子环境（首次运行建议先建）
python - <<'PY'
from scientific_intelligent_modelling.srkit.conda_env_manager import env_manager

# 仅创建核心算法环境
for name in [
    "sim_base",
    "sim_dso",
    "sim_tpsr",
    "sim_e2esr",
    "sim_llm",
    "sim_qLattice",
    "sim_iMCTS",
]:
    env_manager.create_environment(name)
PY
```

> `sim` 为主控环境，`sim_base/sim_tpsr/...` 为子进程算法环境。

---

## 3. 通用 API 示例

```python
from scientific_intelligent_modelling.srkit.regressor import SymbolicRegressor
import numpy as np

rng = np.random.RandomState(0)
X = rng.rand(64, 2)
y = 1.3 * X[:, 0] + 2.0 * X[:, 1] + 0.01 * rng.randn(64)

reg = SymbolicRegressor(
    "tpsr",
    problem_name="demo",
    seed=42,
    force_cpu=True,
)

reg.fit(X, y)
print(reg.get_optimal_equation())
print(reg.get_total_equations()[:3])
print(reg.predict(X[:5]))
```

## 5. TPSR 权重与闭环

`check/check_tpsr.py` 同时验证 `e2e` 与 `nesymres` 两条链路。

- e2e（transformer）权重：默认期望路径 `scientific_intelligent_modelling/algorithms/tpsr_wrapper/tpsr/symbolicregression/weights/model.pt`
  - 缺失时会尝试通过 `symbolicregression_model_url` 自动下载（默认官方链接）。
- NeSymReS 约束：默认需要
  - `nesymres/jupyter/100M/eq_setting.json`
  - `nesymres/jupyter/100M/config.yaml`
  - `nesymres/weights/10M.ckpt` 或 `10MCompleted.ckpt`

运行示例：

```bash
# 建议显式声明路径
TPSR_E2E_MODEL_PATH=.../scientific_intelligent_modelling/algorithms/tpsr_wrapper/tpsr/symbolicregression/weights/model.pt \
TPSR_NESYMRES_MODEL_PATH=.../scientific_intelligent_modelling/algorithms/tpsr_wrapper/tpsr/nesymres/weights/10M.ckpt \
python check/check_tpsr.py
```

## 6. Docker 打包部署（推荐别人直接部署）

目标：在镜像内直接运行本仓库工具，不依赖开发机环境。

### 6.1 准备 Dockerfile

在仓库根目录创建 `Dockerfile`：

```dockerfile
FROM continuumio/miniconda3:24.7.1-0

ENV DEBIAN_FRONTEND=noninteractive \
    PYTHONUNBUFFERED=1 \
    TZ=Etc/UTC

RUN apt-get update && apt-get install -y --no-install-recommends \
    git \
    wget \
    curl \
    ca-certificates \
    procps \
    build-essential \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app
ENV PYTHONPATH=/app:${PYTHONPATH}

COPY . /app

# 安装主控环境（避免在镜像运行时再 pip 源码安装）
RUN conda create -y -n sim python=3.10 pip && \
    conda run -n sim python -m pip install --upgrade pip && \
    conda run -n sim pip install -e .

ENTRYPOINT ["conda", "run", "-n", "sim", "python"]
CMD ["check/check_gplearn.py"]
```

### 6.2 直接构建镜像

```bash
docker build -t sim-toolbox:latest .
```

### 6.3 直接运行容器（推荐）

```bash
# 默认进入 gplearn 闭环（CMD）
docker run --rm -it sim-toolbox:latest

# 执行单条 check
for name in check/check_gplearn.py check/check_tpsr.py; do
  docker run --rm -it \
    -v $(pwd)/artifacts:/app/artifacts \
    -e PYTHONUNBUFFERED=1 \
    sim-toolbox:latest $name
  done
```

### 6.4 带权重文件跑 TPSR 的部署写法

```bash
docker run --rm -it \
  -v $(pwd)/tpsr_weights:/app/artifacts/tpsr_weights \
  -e TPSR_E2E_MODEL_PATH=/app/artifacts/tpsr_weights/symbolicregression/model.pt \
  -e TPSR_NESYMRES_MODEL_PATH=/app/artifacts/tpsr_weights/nesymres/10M.ckpt \
  sim-toolbox:latest check/check_tpsr.py
```

### 6.5 可选：Compose 一键启动

```yaml
version: "3.8"
services:
  sim:
    image: sim-toolbox:latest
    container_name: sim-toolbox
    environment:
      - PYTHONUNBUFFERED=1
      - TPSR_E2E_MODEL_PATH=/app/artifacts/symbolicregression/model.pt
      - TPSR_NESYMRES_MODEL_PATH=/app/artifacts/nesymres/10M.ckpt
    volumes:
      - ./artifacts:/app/artifacts
    entrypoint: ["conda", "run", "-n", "sim", "python"]
    command: ["check/check_tpsr.py"]
```

## 7. 生产部署注意点（建议）

1. 将权重目录 `artifacts/` 作为独立卷挂载，镜像内不必打包大文件。
2. 首次运行时间较长（首次创建子环境 + 下载模型），建议预热。
3. 容器里运行 check 时，如缺少模型会立即报错并给出具体路径提示。
4. 主工具与算法之间隔离在不同 conda 子环境，保留 `toolbox_config.json` 与 `envs_config.json` 的默认映射。

---

## 8. 常见报错处理

- `env not found`：先 `git submodule update --init --recursive`。
- `Cannot find TPSR_E2E_MODEL_PATH`：设置环境变量或放到默认路径。
- `Missing nesymres files`：确认 `config.yaml`、`eq_setting.json` 与 ckpt 均存在。
- `Julia backend` / 首次编译慢：首次启动属正常现象。
- `SerializationError` / `PickleError`：建议先在脚本中检查 `fit -> get_optimal_equation -> get_total_equations -> predict`。
